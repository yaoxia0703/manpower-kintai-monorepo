# 权限管理过滤分页设计

## 目标

将权限管理列表统一改为后端数据库分页查询，并支持按菜单层级和关键词组合筛选。移除原有按菜单返回完整列表的接口分支，避免筛选结果在前端形成非分页数据。

## 页面交互

- 权限管理卡片标题栏保留“权限管理”和“新建”按钮，移除“更新”按钮。
- 查询区沿用“部下管理”页面的表单布局和操作方式。
- 关键词输入框支持按权限 `code` 或 `name` 进行包含式模糊匹配。
- `code` 匹配忽略字母大小写；`name` 使用数据库既有字符排序规则匹配。
- 输入框按 Enter 或点击“搜索”按钮时执行查询，不在每次输入字符时发送请求。
- 菜单使用可清空的树形下拉框。
- 选择父菜单时，结果包含该菜单及全部后代菜单绑定的权限；选择叶子菜单时仅包含该菜单绑定的权限。
- 菜单条件与关键词条件同时存在时使用 AND 组合；`code` 与 `name` 之间使用 OR 组合。
- “重置”按钮清空关键词和菜单条件并查询全部权限。
- 搜索、重置和修改每页条数时将页码重置为第 1 页。
- 普通翻页以及新增、编辑、删除、启用或禁用后的重新加载保留当前筛选条件。

## API 设计

统一使用现有分页资源：

```http
GET /admin/sys/permissions?page=1&size=10&menuId=6&keyword=admin
```

查询参数：

- `page`：页码，默认 `1`。
- `size`：每页条数，默认 `10`，继续遵守公共分页对象的上限约束。
- `menuId`：可选菜单 ID。传入时筛选该菜单及全部后代菜单。
- `keyword`：可选关键词。后端去除首尾空格，空字符串等同于未传。

响应继续使用现有 `PageResult<PermissionResponse>`，其中 `records`、`total`、`page`、`size`、`pages` 全部基于过滤后的数据库查询结果。

旧的 `GET /admin/sys/permissions/by-menu/{menuId}` 不再使用，也不恢复。前端删除 `fetchPermissionsByMenu` 的导入和调用，所有列表加载均通过 `fetchPermissions` 完成。

## 后端数据流

1. Controller 接收 `page`、`size`、可选 `menuId` 和可选 `keyword`。
2. 应用服务在 `menuId` 存在时取得目标菜单及全部后代菜单 ID。
3. 权限分页查询在数据库侧组合菜单 ID 条件与关键词条件。
4. 查询按权限 `sort` 升序排序，并增加 `id` 升序作为稳定的次级排序，避免相同 `sort` 值跨页漂移。
5. MyBatis-Plus 分页插件生成总数和当前页记录，服务转换为 `PageResult` 返回。

菜单不存在时返回空分页结果，不将其解释为无条件查询，避免错误的菜单 ID 意外暴露全部权限。

## 前端数据流

前端维护两组筛选状态：表单中正在编辑的值和已提交的查询值。只有按 Enter、点击“搜索”或点击“重置”时，已提交条件才发生变化。这样用户修改输入框但尚未搜索时，翻页和增删改后的刷新仍使用列表当前展示的查询条件。

`fetchPermissions` 接收：

```ts
{
  page?: number
  size?: number
  menuId?: number
  keyword?: string
}
```

列表始终从响应的 `records` 和 `total` 更新，不再为菜单筛选结果计算本地 `length`。

## 错误处理

- 菜单选项加载失败时显示现有 Element Plus 错误提示，列表查询仍可在无菜单条件下使用。
- 权限分页查询失败时保留当前筛选表单，结束 loading 状态并显示错误提示。
- 关键词仅包含空白时不向后端发送 `keyword` 参数。

## 测试与验收

后端测试覆盖：

- 未传筛选条件时返回正常分页结果。
- 父菜单筛选包含自身和全部后代菜单权限。
- 叶子菜单筛选只包含自身权限。
- 不存在的菜单返回空分页结果。
- 关键词匹配 `code` 或 `name`，并验证 Code 大小写不敏感。
- 菜单与关键词组合筛选后，`records`、`total` 和 `pages` 均基于过滤结果。

前端验收覆盖：

- 类型检查和生产构建通过，不再出现缺少 `fetchPermissionsByMenu` 导出的错误。
- 搜索、Enter、重置、翻页和修改每页条数传递正确参数。
- 筛选后新增、编辑、删除或切换状态仍保留已提交的查询条件。
- 页面没有“更新”按钮，保留“新建”“搜索”“重置”按钮。

## 非目标

- 不增加权限状态、HTTP 方法或 API Path 等额外筛选条件。
- 不修改路由访问权限 `admin:permission:read`。
- 不对其他管理页面进行无关重构。
